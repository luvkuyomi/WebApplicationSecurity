1. SQL Injection

SQL Injection is a vulnerability that allows attackers to interfere with the queries that an application makes to its database. This can lead to to unauthorized access, data, leakage, or even deletion of records.

  • Occurs when user input is embedded into SQL quesries without validation
  • Can expose or destroy sensitive data
  • Example: ' OR 1=1 --

2. Cross-Site Scripting (XSS)

XSS attacks occur when an attacker inject malicious scripts intop content that is then served to other users. These scripts can steal session tokens, redirect users, or manipulate web pages.

  • Types: Stored, Reflected, DOM-based
  • Targets users, not systems
  • Often results from improper input sanitization

Cross-Site Request Forgery (CSRF)

CSRF tricks users into performing actions they diodn't intend on a website where they're authenticated. This can result in unwanted account changes, transactions, or data submissions.

  • Exploits trust a site has in the user's browser
  • Usually delivered via image tags or hidden forms
  • Often mitigated using anti-CSRF tokens

4.Broken Authentication

This happens when untrusted data is used to abuse the logic of an application, inflict denial of service, or execute abitrary code during deserialization.

  • Can lead to remote code execution
  • Triggered by manipulation of seralized objects
  • Commin in apps that use formats like JSON or XML

Threat Impact Mitigation
